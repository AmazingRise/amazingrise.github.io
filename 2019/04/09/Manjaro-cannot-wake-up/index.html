Manjaro 挂起后无法唤醒之解决方案(Dell G7 or Dell XPS)

Posted on 2019-04-09 | Edited on 2019-07-24

前言

前段时间解决了Optimus环境下Manjaro的多显示器问题。（下一篇博文要好好谈谈这个问题）
然而要想给笔记本用上manjaro并不是一帆风顺。刚刚折腾好，这又遇到了长时间挂起后无法唤醒的问题。
两个显示器就在一直闪，迟迟无法进入桌面。tty进不去，REISUB大法也无效了……
其实根源问题还是在Optimus…啊 万恶的NVIDIA…

问题分析

为什么Ubuntu可以正常挂起，而manjaro不行？

  1. 原因在于他们的电源管理方式不同
    之前Ubuntu上也是无法正常挂起的，遇到的问题差不多，也是长时间挂起无法唤醒。
    后来安装了Laptop Mode Tools就没事了~

  2. Ubuntu在适配XPS等戴尔笔记本的时候，做了一些手脚。
    这里有一些bug，但是Ubuntu修复了，而Manjaro这边还需要手动修改才可以。
    具体请各位看解决方案x2
    Update: 这个问题的罪魁祸首是NVIDIA. 如果前面的解决方案不能解决请翻到最后。

解决方案x1

如果是TLP导致的电源问题，请看下面祭出解决方案x1。
戴尔笔记本请在看完这个之后，再看下面的解决方案x2。

最重要的一步，找到这个配置文件：

将 ENABLE_LAPTOP_MODE_ON_AC=0改为=1
这样才能让笔记本在接入AC电源的情况下使用laptop mode tools。

关于Laptop mode tools的具体配置，请参见ArchWiki.
Laptop Mode Tools : Configuration

解决方案x2

Updated on 2019.7.24
Update x2：对于戴尔笔记本(G7或者XPS)，如果这个问题仍然会偶尔出现

如何检查自己的Dell是否中枪

终端里面输入：cat /sys/power/mem_sleep
如果内容是：[s2idle] deep，那么恭喜，这个自动生成的设置是错误的。

如何解决

  1. 编辑/etc/default/grub，然后找到GRUB_CMDLINE_LINUX。
    在双引号里面追加这一参数：mem_sleep_default=deep
    举个例子：（省略号代表之前已有的参数）

（注意新的参数要和前面的参数保持一个空格的距离）

  1. 运行sudo update-grub

  2. 重启后检查一下，如果你的修改是正确的话
    那么这次cat /sys/power/mem_sleep的结果应该是：s2idle [deep]
    （注意方括号位置的移动。）
    这样就成功修改了。

注：我给出的方案来自Dell XPS 15 9570 with PRIME -> OS Freezes on wake up。理论上来讲添加这一个参数就够。

注2：而Dell Community 里面给出的解决方案中的参数和我给出的略有不同。
社区里面给的比我这里多了两个，读者可以参考一下。

罪魁祸首

为了满足大家的好奇心我还是把这一节补上了~
据说 Dell 的主板的某些地方会「误导」linux内核选择正确的睡眠方式。
而 Ubuntu 那边已经针对Dell机器针对性地修复了这个bug。
（具体原理参见参考文章1）

这就导致了虽然更换了Laptop mode tools， 问题仍然会时不时地复发。
不过不亏，因为Laptop mode tools提供了更多的电源管理方式，个人认为要比TLP强一些
