Manjaro 挂起后无法唤醒之解决方案

Posted on 2019-04-09

前言

前段时间解决了Optimus环境下Manjaro的多显示器问题。（下一篇博文要好好谈谈这个问题）
然而要想给笔记本用上manjaro并不是一帆风顺。刚刚折腾好，这又遇到了长时间挂起后无法唤醒的问题。
两个显示器就在一直闪，迟迟无法进入桌面。tty进不去，REISUB大法也无效了。
我的笔记本上还安装了Ubuntu，而Ubuntu是可以正常挂起的。（对比一下问题一目了然2333）
于是结合官方维基以及官方论坛上的说法，我成功解决了这个问题。现将解决方案分享如下。

正文

下面我将对这个问题进行分析。具体解决方案请翻到最后。

为什么Ubuntu可以正常挂起，而manjaro不行？

原因在于他们的电源管理方式不同。其实之前Ubuntu上也是无法正常挂起的，遇到的问题差不多，也是长时间挂起无法唤醒。
后来采用了”laptop mode“才得以正常休眠。
而Manjaro默认的电源管理是TLP。
详细了解请戳这里：TLP - ArchWiki。
所以如果我们把默认的电源管理换成以前的laptop mode tools也许就能解决问题。
不知道为啥TLP问题这么多。

Laptop Mode Tools为何方神圣？

引用自官方维基

Laptop Mode Tools is a laptop power saving package for Linux systems. It is the primary way to enable the Laptop Mode feature of the Linux kernel, which lets your hard drive spin down. In addition, it allows you to tweak a number of other power-related settings using a simple configuration file.

值得注意的是, Laptop Mode Tools与原有的TLP相冲突。两者不能共存
所以我们就有思路解决这个问题啦～

解决方案

各位看官别急。下面祭出解决方案。

首先，关掉原来自带的TLP电源管理。

$ systemctl disable tlp
$ systemctl disable tlp-sleep.service

然后安装Laptop Mode Tools：

$ sudo pacman -S laptop-mode-tools

开启服务！

$ sudo systemctl enable laptop-mode.service

大功告成！

关于Laptop mode tools的具体配置，请参见ArchWiki.
